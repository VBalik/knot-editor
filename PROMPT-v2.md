> Документ передачи для новой сессии. Проект в основном работает; не решена

## 1. Продукт (готов, менять только физику)

Одностраничное приложение `knot_editor.html` (один файл, three.js r128 с CDN,
без сборки). Слева 2D-редактор: рисование замкнутой кривой мышью, сглаживание
Catmull-Rom, поиск самопересечений, клик по серому кольцу меняет над/под
(классическая отрисовка: верхняя прядь сплошная, нижняя с малым разрывом внутри
кольца). Справа 3D: трубка TubeGeometry, пол с мягкой тенью, орбитальная камера,
узел всегда в центре. Компактные тулбары одной строкой; статус-плашка в правом
нижнем углу 3D. Кнопки: ▶ Физика, OBJ (экспорт), Телем (телеметрия для отладки).
Ползунки: Жёсткость 0–30, Отталкивание 0–20 (после равновесия их сдвиг сам
перезапускает релаксацию). Тип узла определяется **определителем диаграммы**
(Александер при t=−1) и показан чипом: «тривиальный» (det=1) или «узел · det N».

## 2. Жёсткие инварианты физики (из исходного ТЗ, все обязательны)

1. **Самопересечение струны невозможно никогда** — топология сохраняется при
   любых настройках (проверяется определителем 3D-кривой, см. §6).
2. **Никаких паразитных динамических эффектов**: ни автоколебаний, ни бегущих
   волн, ни «спиралей»/гофры на струне, ни дрожания, ни вечного вращения/дрейфа.
3. Узел не улетает и не схлопывается ни при каких ползунках.
4. **Тривиальный узел (det=1) обязан быстро прийти в точную окружность**
   (разброс радиуса < ~1%). Любое скручивание (writhe) свободно раскручивается.
5. Настоящий узел (det>1) приходит в гладкую объёмную «каноническую» форму и
   **останавливается** (равновесие объявляется, статус меняется).
6. Сегменты абсолютно жёсткие (длина каждого = L0, полная длина постоянна);
   кручения в модели нет вообще (только позиции вершин).
7. Overdamped: никаких скоростей и инерции.

## 4. Текущая физика (коммит fe128be) и её известный дефект

Ядро: **монотонный проекционный градиентный спуск** по энергии
E = изгиб(Σθ²) + мягкое дальнее отталкивание + кубический барьер толщины
(D=1.6·L0), на многообразии жёстких рёбер (`edgeProject`, Якоби + микро-рескейл).
Направление: соболевски сглаженный градиент (12 проходов усреднения) →
касательная проекция (`dirTangentProject`); запасной ярус — чистый проецированный
градиент. Backtracking line search: шаг принимается только при уменьшении E
после проекции. Анти-туннель: единый кап шага 0.35·(мин. зазор) + барьер +
контроль зазора при приёмке. Аналитический градиент сверен с конечными
разностями (1e-4). Фаза выбирается определителем: det=1 → без отталкивания
(чистое распрямление → круг), det>1 → полная энергия.

**Что проверено зелёным** (эмуляция кадров + Python-определитель): клякса →
окружность (rvar 0.97%); трилистник/восьмёрка 5/5 и трилистник 30/20 — гладкий
канон, равновесие за ~26 кадров; кейс det=15 (раньше вечно осциллировал) —
энергия монотонно замерла, форма гладкая, det=15 сохранён в 9 проекциях;
~4 мс/шаг на дефолте.

**Известный дефект (последняя жалоба, телеметрия утеряна): «узел вибрирует и
релаксации не происходит».** Главная гипотеза: цикл «planarize-пинок ↔ спуск»
в фазе 1 — эскейп из сёдел замкнутой эластики реализован грубым пинком
`planarizePass(0.6)`, который ПОВЫШАЕТ энергию вне монотонной схемы; если
конфигурация не выходит к кругу, пинки повторяются каждый кадр = видимая вечная
вибрация без сходимости. Вторая гипотеза: в фазе 2 при полном отклонении шага
(line search не нашёл спуска из-за жёсткого барьера) состояние восстанавливается
из снапшота, но вспомогательные операции (recenter) дают микросдвиг — кадр за
кадром это читается как дрожание, а settle-критерий по maxMove не срабатывает.
Диагностировать по телеметрии: physPhase, монотонность bendEnergy, поведение
maxMove/rvar хвоста ряда.

## 5. КАТАЛОГ ГРАБЛЕЙ — испробовано и провалилось (не повторять!)

1. **Конвейер независимых проходов** (изгиб → отталкивание → коллизия →
   ресэмплинг → рескейл → снятие вращения): проходы воюют, на сложных узлах при
   высоких настройках — устойчивые предельные циклы (авто-колебания maxMove
   0.16–0.42·L0 вечно) и заторы. Принципиально не чинится дозировками.
2. **Пер-вершинные ограничители шага** (клампы по клиренсу): соседним вершинам
   достаются разные лимиты → гладкое поле движения пилится в коротковолновую
   гофру → «спирали» на струне (это ИЗГИБ-НАКОПЛЕНИЕ от сжатия, кручения в
   модели нет). Ограничивать шаг можно только ЕДИНЫМ масштабом на весь узел.
3. **Rescale к полной длине как обратная связь**: гофра добавляет длину →
   глобальное сжатие → теснее контакты → жёстче клампы → больше гофры (разгон).
   Полную длину держать проекцией рёбер, не масштабом пространства.
4. **Туннелирование**: (а) большой шаг сквозь малый зазор; (б) коллизия,
   расталкивая одну пару, вдавливает вершины в третью прядь; (в) ресэмплинг
   срезает углы хордами сквозь соседнюю прядь; (г) при инициализации лифта
   «почти-касания» (не пересечения в проекции) стартуют с зазором ~0.03·L0.
   Лечение, которое работает: вершины лифта ТОЧНО в проходах (якоря ±H),
   раскрытие контактов коллизией до старта, единый кап шага от минимального
   зазора, приёмка шага с контролем зазора.
5. **Классификация узла по физике** (самокасание как признак узла) — путает
   writhe с узлом. Только топологический инвариант: определитель диаграммы.
   Он же ловит порчу диаграммы (было: кластеризация пересечений по пикселям
   удаляла настоящее пересечение — дедуп только по паре дуговых позиций!).
6. **Жёсткая пружина растяжения в энергии** вместо проекции: число
   обусловленности ~100:1 делает явный спуск бесполезно медленным.
7. **Голый градиентный спуск по изгибу**: жёсткость коротких волн (CFL) требует
   ~N² шагов до окружности. Обязательно сглаживание градиента (соболевское
   предобуславливание) + касательная проекция направления, иначе изгибный
   градиент (он почти весь «нормальный», хочет укорачивать кривую) гасится
   проекцией рёбер и line search замирает.
8. **Сёдла замкнутой эластики** (волнистые замкнутые равновесия Σθ²): строгий
   спуск в них честно останавливается, до круга не доходит. По Лангеру–Зингеру
   устойчива только окружность — эскейп обязателен, но он должен быть
   СОГЛАСОВАН с монотонной схемой (текущий planarize-пинок — нет; см. §4).
9. Не-физика: GitHub Pages собирает старый коммит без принудительного POST;
   headless-вкладка превью замораживает rAF и ресайзится в 0×0 (тесты физики —
   только прямыми вызовами relaxStep/эмуляцией кадров, НЕ через animate);
   `.gitignore` уже содержит `knot-telemetry*.json`.

## 6. Диагностический инструментарий (готов, использовать всегда)

- **Кнопка «Телем»** выгружает JSON: meta (N, L0, ползунки, фаза, det),
  verts (вся 3D-ломаная), diagram (smooth-кривая + пересечения с over/under),
  series (до 400 кадров: maxMove, omega, minSegL0, maxCurv, bendEnergy, rvar,
  drift). Пользователь кладёт файл в `Knots/` — читать оттуда.
- **Python-определитель** (эталонная проверка топологии 3D-кривой): алгоритм —
  Александер при t=−1: дуги между последовательными подпроходами; на каждое
  пересечение строка (+1 вход, +1 выход, −2 над-дуга, совпадения складываются);
  минор без последней строки/столбца; |целочисленный det| через Fraction.
  Экстракция из ломаной: проекция → пересечения несоседних отрезков
  (полуинтервалы [0,1)!) → over по глубине. ОБЯЗАТЕЛЬНАЯ валидация перед
  использованием: трилистник=3, восьмёрка=5, окружность=1, (1,5)-виток
  (writhe)=1; считать в 3 проекциях + случайные повороты — значения обязаны
  совпасть. Скрипт пересоздавать в скретчпаде (tmp чистится между сессиями).
- Внутри страницы: `knotDiagnostics()`, `minSegGap()`, `curveRvar()`,
  `energyGrad(false).E` — для проверки монотонности.

## 7. Критерии приёмки (полный набор, гонять после КАЖДОЙ правки физики)

| Тест | Требование |
|---|---|
| Клякса/лемниската/writhe-розетка (det=1) | окружность rvar<1.2%, равновесие объявлено, без вибрации |
| Пресеты 3₁/4₁ (det 3/5), 5/5 | гладкий канон (maxCurv<0.3), zSpan>1.2, равновесие ≤ ~50 кадров |
| Трилистник 30/20 (максимум ползунков) | то же, без гофры и колебаний |
| Сложные рисунки: кейсы det=7 (22 пересечения), det=11 (12), det=13 (16), det=15 (11) | det сохраняется весь прогон (Python, 3 проекции), равновесие объявляется, форма гладкая |
| Монотонность | энергия strictly ↓ до равновесия (хвост series без колебаний) |
| Смена ползунка после равновесия | релаксация перезапускается и снова замирает |
| Производительность | ≤ ~10 мс/шаг на дефолте при N≈200 |

## 8. Рекомендации к новой попытке (выбор за новой сессией)

Текущее энергетическое ядро — правильная основа (монотонность = отсутствие
колебаний по построению). Его надо ДОВЕСТИ, а не выбрасывать:
(а) заменить planarize-пинок на **монотонный сэддл-эскейп**: пробные малые
возмущения (несколько случайных гладких направлений / направление наименьшей
кривизны энергии), принимаемые только если через K шагов спуска E стала НИЖЕ
исходной, иначе откат; (б) убрать источники несогласованных микросдвигов при
отклонённом шаге (recenter только после принятых шагов); (в) settle-критерий:
«шаг line search → 0 И (det=1 ? rvar<1.2 : форма стабильна по горизонту)» —
с явным сообщением, если достигнут локальный минимум-не-круг; (г) по желанию
ускорение: momentum/L-BFGS по сглаженному градиенту с тем же монотонным
контролем. Альтернатива, если доведение забуксует: Discrete Elastic Rods
(Bergou et al.) с вязким демпфированием и адаптивным dt, или связка
«curve-shortening + SONO» с приёмкой по энергии. Любой вариант обязан пройти §7.
